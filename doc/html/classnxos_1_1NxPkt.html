virtual unsigned short nxos::NxPkt::getPkt ( unsigned char **  pkt)
pure virtual

Gets a pointer to the Packet being constructed to be sent or received by the switch. For IP Packets, to be sent (or) received by this switch, ignore the ethernet header as it contains Source MAC and Dest MAC as 0. As the lower network Layer strips the Ethernet header on receive and on send it attaches the right Ethernet header automatically based on the provided next-hop or eggress interfaece.

virtual void nxos::NxPkt::sendIpPkt ( char *  eggress_interface = NULL,
char *  context_name = "default" 
)
pure virtual

Send the IP[v6] packet. Ethernet header is automatically added by the underlying network stack based on the eggress_interface if provided. If not its resolved based on the dest_ip[v6].

Parameters
pktObject to the constructed/built Ip packet to send.
eggress_interfaceOptional. Interface on which the packet needs to be sent out. Default is NULL hence the switch resolves the outgoing interface.
context_nameOptional.
Returns
returns a bool. True if the packet was accepted by the underlying Network stack layer to send it out. It does not gaurantee that the packet is sent on the wire. Write your own tx/rx procotol to ensure packet delivery..
Exceptions
Throwsa string exception on error.
See Also
All Sent Packets are stored in a pcap file located at /nxos/<appname>_pkt_sent.pcap
