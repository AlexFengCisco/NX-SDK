Detailed Description

Policy class to hold information about Filters. Filters can then added to other objects that requires filtering based on policies. For example) Filters can be added to NxPktMgrImpl to filter certain packets only.

Creates a policy entry to add a set of match and action filters to it. Each entry is associated with a seqeuence id. Policy Enteries are evaluated starting from the lowest sequence ids.

Parameters
seq_iddictates the order of filtering to be for this policy.
match_allOptional. If set to TRUE then all match statements should succeed for the given data for the entry to match. If a match is not applicable to the passed data then it is ignored (implicit match) and is moved to the next match. If set to FALSE then if anyone of the matches succeed then the entry is said to be a match. Default is FALSE
